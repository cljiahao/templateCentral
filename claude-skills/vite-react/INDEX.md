# Vite + React Skills

All skills for Vite + React projects scaffolded from templateCentral.

| Skill | Description |
|-------|-------------|
| `scaffold/` | Use when the user wants a lightweight React SPA without SSR, or needs a Vite + React + TypeScript starter with React Router and TanStack Query. |
| `code-standards/` | Use when writing or reviewing any code in a Vite + React project — covers file naming, export style, function vs const, component placement, and performance rules. |
| `add-feature/` | Use when adding a new domain area (e.g., projects, auth, analytics) that needs its own components, hooks, API services, and types under src/features/. |
| `add-page/` | Use when adding a new URL route, creating a page component, or the user asks to add a page to a Vite + React SPA. |
| `add-component/` | Use when creating a new React component and unsure whether it belongs in features/, widgets/, or layout/, or when adding a shared component with barrel exports. |
| `add-integration/` | Use when connecting to an external API (e.g., GitHub, Stripe, OpenAI) from a Vite + React SPA and need a typed client, Zod response schemas, and service layer. |
| `add-test/` | Use when adding tests for components, hooks, or services — covers Vitest unit tests, Testing Library component tests, and React Query hook tests. |
